Galatiq Invoice Agent - Overview

What We Created
1) End-to-end pipeline orchestrated by LangGraph.
2) Parsers for TXT, JSON, CSV, XML, and PDF invoices.
3) Structured schema output with validation and approval.
4) Fraud rules module with hot-reloaded risky vendor list.
5) Structured JSON logging for each stage.

How The Agent Works (Flow)
1) Ingestion
   - Detects file type and parses invoice content.
   - Extracts: invoice_id, vendor, amount, currency, due_date, items.
   - Adds: notes, suspicious_flags, missing_fields, parsing_confidence.
   - If Grok API key is set later, it can enrich extraction.

2) Validation
   - Checks inventory (SQLite).
   - Flags unknown items, out-of-stock, stock exceeded, invalid quantities.
   - Marks missing/uncertain fields as "review".
   - Adds a status_reason.

3) Approval
   - Fail -> Reject payment.
   - Review -> Route to manual review.
   - Pass -> Approve unless amount is too high or risk flags exist.
   - Outputs a clear action + reason.

4) Payment
   - If approved, calls mock payment.
   - Otherwise, skips payment.

Fraud Detection
- High total amounts.
- Risky vendors (configurable in data/risky_vendors.txt).
- Duplicate invoices (tracked in data/fraud_seen.jsonl).
- Urgent or wire-transfer language.
- Suspicious item names (fake/fraud/scam).

Key Files
- main.py: entrypoint
- agents/langgraph_flow.py: LangGraph orchestration
- agents/ingestion.py: parsing and post-processing
- agents/validation.py: inventory validation
- agents/approval.py: approval logic
- agents/payment.py: mock payment
- schema.py: data models
- utils/fraud.py: fraud rules
- utils/logging.py: JSON logs
- utils/schemas.py: pydantic schemas

How to Run
python main.py --invoice_path data/invoices/invoice_1001.txt
